# Navigation — the single go_router, refreshListenable, navigate-by-ID

Grounds `docs/flutter/05-navigation.md`. **Decision:** `go_router` (first-party) + type-safe routes
via `go_router_builder` (`@TypedGoRoute` + `GoRouteData`). One `StatefulShellRoute.indexedStack` shell
for ~6 top-level modules, full-screen flows above it via `rootNavigatorKey`, declarative `redirect`
guards fed by a Riverpod `refreshListenable`, `restorationScopeId` end-to-end. Notification payloads
map to screens through a **pure `payload → location` function**. Rejected: auto_route (single-maintainer),
raw Navigator 2.0, named routes, Beamer.

## Folder layout

```text
apps/car_and_pain/lib/src/routing/
  app_router.dart          # the one GoRouter + rootNavigatorKey + per-branch keys
  routes.dart              # @TypedGoRoute / GoRouteData declarations
  routes.g.dart            # generated by go_router_builder (git-ignored, CI-regenerated)
  notification_router.dart # PURE payload -> location mapper (Flutter-free where possible)
  guards.dart              # pure redirect helpers (onboarding, has-vehicle)
```

## The single GoRouter

```dart
final _rootNavigatorKey = GlobalKey<NavigatorState>(debugLabel: 'root');
final _dashboardKey     = GlobalKey<NavigatorState>(debugLabel: 'dashboard');
// …one navigatorKey per branch

GoRouter buildAppRouter({
  required Listenable refresh,      // fed by Riverpod onboarding/vehicle-count state
  required String initialLocation,  // from getNotificationAppLaunchDetails(), else '/'
}) => GoRouter(
      navigatorKey: _rootNavigatorKey,
      initialLocation: initialLocation,
      restorationScopeId: 'app_router',
      refreshListenable: refresh,     // re-runs guards when Riverpod state changes
      redirect: appRedirect,          // pure function
      routes: $appRoutes,             // generated aggregate from routes.g.dart
    );
```

```dart
MaterialApp.router(
  restorationScopeId: 'app',
  routerConfig: appRouter,
  locale: appLocale,               // app-controlled locale drives Directionality/RTL
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: AppLocalizations.supportedLocales,
);
```

## refreshListenable ← Riverpod

The guard surface is tiny (onboarding-complete, has ≥1 vehicle), so make it reactive by feeding a
Riverpod-backed `Listenable` into `refreshListenable`. A `ChangeNotifier` bridged from the onboarding/
vehicle-count providers re-runs `appRedirect` whenever that state changes.

```dart
String? appRedirect(BuildContext context, GoRouterState state) {
  final loc = state.matchedLocation;
  if (!onboardingComplete && loc != '/onboarding') return '/onboarding';
  if (noVehiclesYet && needsVehicle(loc) && loc != '/vehicles/new') return '/vehicles/new';
  return null;
}
```

Keep `redirect` **pure and idempotent**, and always exclude the target location (`loc != '/onboarding'`)
or it loops forever.

## Tab shell

```dart
StatefulShellRoute.indexedStack(
  builder: (context, state, shell) => ScaffoldWithNavBar(navigationShell: shell),
  branches: [
    StatefulShellBranch(navigatorKey: _dashboardKey, routes: [ /* Dashboard */ ]),
    StatefulShellBranch(navigatorKey: _vehiclesKey,  routes: [ /* Vehicles master-detail */ ]),
    // Reminders, Costs/TCO, Reports/Charts, Settings …
  ],
);
// switch tabs without rebuilding the branch; re-tap resets to branch root:
navigationShell.goBranch(index, initialLocation: index == navigationShell.currentIndex);
```

`indexedStack` keeps each tab's navigation stack **and scroll position** alive. ~25 feature folders
map onto ~6 shell branches (Dashboard, Vehicles, Reminders, Costs/TCO, Reports/Charts, Settings).

## Navigate by ID — path params, never `extra`

Detail screens carry **stable DB IDs in the path** so any screen is reconstructable from its URL
alone:

```text
/vehicles                                   (list)
/vehicles/:vehicleId                        (detail)
/vehicles/:vehicleId/reminders/:reminderId  (deep-linkable target)
```

```dart
@TypedGoRoute<VehicleDetailRoute>(
  path: '/vehicles/:vehicleId',
  routes: [TypedGoRoute<ReminderDetailRoute>(path: 'reminders/:reminderId')],
)
class VehicleDetailRoute extends GoRouteData with _$VehicleDetailRoute {
  const VehicleDetailRoute({required this.vehicleId});
  final String vehicleId;
  @override
  Widget build(BuildContext context, GoRouterState state) =>
      VehicleDetailScreen(vehicleId: vehicleId);
}

// Navigate with compile-time safety, cross-feature-safe (no feature import):
const VehicleDetailRoute(vehicleId: id).go(context);
```

Prefer `.go()` over `.push()` — since go_router 8.0, imperative `push()` no longer updates the
location redirects/URL observe. Full-screen flows (add/edit, backup/restore, onboarding) set
`parentNavigatorKey: _rootNavigatorKey` so they render **above** the tab bar.

## Notification deep-link bridge (the highest-value pure seam)

```dart
// notification_router.dart — pure, no BuildContext, no plugins
String mapNotificationPayload(String? payload) {
  final p = NotificationPayload.tryParse(payload);
  return switch (p) {
    ReminderPayload(:final vehicleId, :final reminderId) =>
      '/vehicles/$vehicleId/reminders/$reminderId',
    _ => '/', // unknown/legacy payload falls back to dashboard
  };
}
```

```dart
// In bootstrap(), BEFORE runApp — cold start:
final launch = await flnp.getNotificationAppLaunchDetails();
final initial = (launch?.didNotificationLaunchApp ?? false)
    ? mapNotificationPayload(launch!.notificationResponse?.payload)
    : '/';

// In the FLN init callback — warm taps:
onDidReceiveNotificationResponse: (resp) => appRouter.go(mapNotificationPayload(resp.payload));
```

Because the target route needs only the IDs, the screen rebuilds its data from the encrypted SQLite
DB — surviving reboot/Doze/cold-start where `extra` would be null. Notifications are the app's real
deep links, so path-param identity is non-negotiable.

## Rules

- Carry deep-linkable identity in **path params**; never rely on `state.extra` for anything a
  notification, deep link, or restore reaches (null on cold start).
- Prefer `.go()` / typed `.go()`; don't `push()` for shell navigation.
- Keep `redirect` pure and idempotent; always exclude the target location.
- Set `parentNavigatorKey: _rootNavigatorKey` on every full-screen flow; assign explicit per-branch
  `navigatorKey`s.
- Use go_router only for screen-like, deep-linkable destinations; route ephemeral UI (dialogs,
  pickers, snackbars) through plain `showDialog`/`showModalBottomSheet`.
- Guard `context` across async gaps (`if (!context.mounted) return;` or capture `GoRouter.of(context)`
  before the `await`).
- Do NOT add `FlutterDeepLinkingEnabled`/App-Links config — routing is Dart-driven from notification
  payloads, not OS deep links; adding it invites handler conflicts.
- RTL is not the router's job — locale/Directionality live at `MaterialApp`. Only *custom* transitions
  must read `Directionality.of(context)`; a hard-coded `Offset(1,0)` slides the wrong way in fa/ar/ckb.
- CI runs `build_runner` + `build_verify` so typed-route codegen never drifts.

## Testing

- **Pure mapper (highest value):** table-driven tests on `mapNotificationPayload` — valid, malformed/
  legacy, null → `/`. This is the reboot-safe-reminder guarantee.
- **Pure redirect:** `(app-state, location) → expected redirect`; assert no loops.
- **Router in isolation:** build the `GoRouter`, call `router.go(uri)` / typed `Route(...).go()`,
  assert `router.routerDelegate.currentConfiguration` / `router.state.uri`.
- **Cold-start / deep link:** construct with `initialLocation` set to a mapped location, seed an
  in-memory Drift DB, assert the target renders with data rebuilt via the `extra == null` path.
- **Shell (widget test):** tap across `StatefulShellRoute` tabs; verify each branch keeps its stack/scroll.
- **RTL:** force a fa/ar/ckb locale (or `Directionality(textDirection: rtl)`); golden the shell.
- **State restoration:** `tester.restartAndRestore()` to confirm last route + form fields return.
